# PH2-09 隔离式真实测量分页引擎开发计划

## 1. 文档定位

本文件只服务任务 `PH2-09 精细分页算法（二次推进）` 下的子步：

- `PH2-09-dom-measure-engine-plan-v1`
- `PH2-09-dom-measure-engine-v1`

本文件用于沉淀“新增一套与现有分页引擎完全隔离、前端可切换的 DOM 真实测量分页引擎”的实施计划、边界和回退策略。

本文件不是项目通用技术基线，不替代以下项目级文档：

- `LAYOUT3.0产品开发文档.md`
- `功能设计及需求.md`
- `结构化文档模型V1方案.md`
- `progress/roadmaps/PH2-09-分页算法开发线路.md`

## 2. 目标

本轮目标不是改良 `max-fill-v1`，而是新增一套隔离的 `dom-measure-v1`：

- 保留当前 `max-fill-v1` 不变，用户可在前端切换。
- 新引擎输出仍保持 `PageLayout[]`，不推翻预览层和导出层协议。
- 第一版只做单栏真实测量分页主链路。
- 通过隐藏测量容器逐步把分页判断从“估算能放多少”切到“真实测到哪里刚好放下”。

一句话原则：

先把第二引擎独立站起来，再逐步提高它的测量精度和块类型覆盖。

## 3. 规则抽象

### 3.1 页面规则

- 每页是固定页面，不是长滚动容器。
- 继续复用 `ResolvedStyleContract` 的页面宽高、页边距、页眉页脚和正文区尺寸。
- 输出仍是 `PageLayout[]`，保证画布预览、目录页码回填和 HTML/PDF 导出不需要第二套页面模型。

### 3.2 测量规则

- 第一层：继续复用现有顶层块高度测量和真实换行测量。
- 第二层：新增片段级测量缓存接口，专门服务 `dom-measure-v1`。
- 第三层：当新引擎缺少精确片段数据时，先给出保守分页结果，再触发测量回填和自动重排。

### 3.3 切分页规则

- 标题/段落：优先使用真实换行索引做安全切分。
- 列表：优先按项分页，超长列表项再按文本片段切分。
- 表格：优先按行分页，第一版不做表头重复和单元格内部复杂跨页。
- 图片：第一版按整块分页，放不下就换页。
- `/pagebreak`：继续强制换页。
- 多栏：V1 明确不做真实测量多栏重排，后续归 V2。

## 4. 当前边界

### 4.1 本步纳入

- 新增独立算法 ID `dom-measure-v1`。
- 在算法注册表中注册第二个分页引擎。
- 保留前端“分页算法”切换入口。
- 新增独立目录承载真实测量引擎实现。
- 接入单栏真实测量分页主循环。
- 长文本安全切分。
- 列表按项分页。
- 表格按行分页。
- 图片整块分页。
- 运行时缓存与基础回归验证。

### 4.2 本步明确不纳入

- 不覆盖、删除或重写 `max-fill-v1`。
- 不改 `.layout` 文件结构。
- 不做两栏/三栏真实测量分页。
- 不做复杂图片绕排真实测量。
- 不做表头重复、合并单元格复杂跨页。
- 不做导出专用第二引擎。
- 不做长期学习、云同步或新的分页持久化字段。

### 4.3 后续归属

- 两栏/三栏真实测量与栏平衡：归属 `PH2-09` 后续子步。
- 图片绕排与更复杂图文混排：归属 `PH2-10` / `PH2-09`。
- 表头重复和更高保真表格跨页：归属 `PH2-09` / `PH2-10`。
- 孤寡行控制、标题挂靠精修和导出终态高保真对照：归属 `PH2-09` 深化子步。

## 5. 实施顺序

### 5.1 `PH2-09-dom-measure-engine-skeleton-v1`

新增目录：

- `src/engine/typesetting/algorithms/domMeasure/`

建议文件：

- `index.ts`
- `types.ts`
- `pageBuilder.ts`
- `textSplit.ts`
- `tableSplit.ts`
- `measurementCache.ts`

原则：

- 不把新逻辑继续堆进 `estimatedMaxFill.ts`。
- 旧引擎与新引擎目录级隔离。

### 5.2 `PH2-09-dom-measure-algorithm-registration-v1`

- `algorithmIds.ts` 新增 `dom-measure-v1`。
- `registry.ts` 注册第二个算法。
- `styleSettings.ts` 与 `styleSlice.ts` 放开算法白名单。
- 右侧“分页算法”下拉显示中文名，例如“真实测量分页引擎”。

### 5.3 `PH2-09-dom-measure-runtime-state-v1`

运行时新增但不进入 `.layout`：

- `measuredFragmentHeights`
- `paginationMeasurementJobs`
- `paginationMeasurementEpoch`

原则：

- 这些状态只存在于 store、hook 或组件运行态。
- 不污染文档模型真相源。

### 5.4 `PH2-09-dom-measure-canvas-measurement-v1`

扩展隐藏测量层：

- 保留旧的顶层块测量逻辑，继续服务 `max-fill-v1`。
- 新增片段测量任务渲染区，只在新引擎启用时工作。
- 稳定 key 建议使用：`blockId + fragmentRange + contractSignature + blockStyleSignature`。

### 5.5 `PH2-09-dom-measure-fast-pass-v1`

第一轮先做快速粗排：

- 整块优先尝试放入当前页。
- 已知真实换行的文本块优先按行切分。
- 只有边界不确定时才派发片段测量任务。

目的：

- 避免一切换新引擎就触发整篇全量重测。

### 5.6 `PH2-09-dom-measure-append-check-v1`

主循环采用网页式分页规则：

1. 初始化当前页可用高度。
2. 逐块尝试追加。
3. 每次追加候选片段后，使用真实高度校验是否溢出。
4. 溢出则回退，写入当前页，开启新页。

这是 `dom-measure-v1` 的核心行为。

### 5.7 `PH2-09-dom-measure-text-split-v1`

- 优先复用现有 `measuredTextLineBreaks`。
- 保持现有 `TextRun` 切分能力，但逻辑在新引擎内部维护。
- 切分结果继续输出运行时 `LayoutBlock` 片段，避免预览层增加第二种渲染协议。

### 5.8 `PH2-09-dom-measure-list-table-v1`

- 列表先按项装页，再对超长项做文本切分。
- 表格先按行装页。
- 第一版只追求稳定和隔离，不先冲高难度。

### 5.9 `PH2-09-dom-measure-image-v1`

- 使用当前图片真实可见高度。
- 整块放不下就换页。
- 不接复杂图片绕排测量。

### 5.10 `PH2-09-dom-measure-ui-switch-v1`

- 分页策略面板增加引擎选择。
- 中文说明文案区分新旧引擎。
- 必要时在预览区显示轻提示，例如“真实测量分页正在收敛”。

## 7. 风险与回退

### 7.1 当前主要风险

- 当前 `PageLayout` 仍是 `blocks[]`，所以新引擎必须输出运行时块片段，不能发明第二套页面渲染协议。
- 当前隐藏测量层主要量顶层块；片段测量任务是这条路线里最大的新增点。
- 如果把两栏真实测量硬塞进 V1，返工概率很高。

### 7.2 回退策略

- 任何时候都能把算法切回 `max-fill-v1`。
- 新引擎不写入 `.layout`，因此回退只影响运行时状态。
- 旧引擎仍是默认和兜底算法。

## 8. 验证要求

必须覆盖：

- `npm run typecheck`
- `npm run smoke:m1`
- `npm run smoke:m2`
- `npm run build`

本轮至少验证这些场景：

- 单栏长段落跨页。
- 列表长项跨页。
- 表格按行跨页。
- 图片整块换页。
- 新旧引擎切换后页面能稳定重算。
- 切回 `max-fill-v1` 后行为不回退。

## 9. 当前结论

`dom-measure-v1` 的第一步不是追求一次做完，而是先做到：

- 工程里真实存在第二分页引擎。
- 前端能切换。
- 单栏可运行。
- 回退清晰。
- 后续片段测量、多栏和表格深化有明确承接位置。

后续所有真实测量分页小步，都优先沿这个计划推进，不再把新逻辑回堆到 `max-fill-v1` 里。
