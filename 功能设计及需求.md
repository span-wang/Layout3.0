# 项目目标
**产品定位**

这是一款面向笔记、教辅、试卷、讲义、文档资料的「Markdown 驱动 + 可视化排版」工具。

核心价值不是单纯 Markdown 编辑器，也不是传统 Word，而是：

- 用 Markdown 快速输入内容
- 自动解析为结构化文档
- 自动完成分页、目录、页眉页脚、样式套用
- 支持在预览页上直接微调
- 最终导出为 PDF、图片或可再次编辑的工程文件

产品关键词：

- Markdown 输入
- 结构化 JSON
- 分页渲染
- 所见即所得编辑
- 模板化排版
- 教辅/试卷/笔记场景扩展
- AI 辅助生成与排版

---

## 一、目标用户与核心场景

### 1. 目标用户

- 教师：制作讲义、试卷、答案解析、课堂笔记
- 教培机构：批量生产教辅内容、练习册、资料包
- 学生：整理笔记、错题集、复习资料
- 内容创作者：制作知识手册、PDF 教程、课程资料
- 编辑/排版人员：将结构化内容快速转换为正式版式

### 2. 高频使用场景

- 将一篇 Markdown 讲义快速排成 PDF
- 将题目、解析、答案自动排成试卷格式
- 将课堂笔记整理成带目录、页码、标题层级的学习资料
- 批量套用统一机构风格模板
- 对自动排版结果进行少量手动调整
- 导出适合打印或分发的 PDF


# 主链路

导入/新建文档 → Markdown 编辑/解析 → 结构化文档模型 → 自动排版 → 分页预览 → 可视化微调 → 样式模板套用 → 校验 → 导出/保存
### 关键说明

- Markdown 是内容输入层
- JSON 是中间结构层，不直接暴露给普通用户
- 分页渲染是排版引擎层
- 预览编辑是可视化调整层
- 模板是样式复用层
- 导出是交付层

建议将产品分成三个工作区：

- 左侧：文件管理 / 大纲 / 搜索结果
- 中间：分页预览画布
- 右侧：属性面板 / 样式面板 / 页面设置 / AI 面板
- 顶部：全局工具栏，如保存、撤销、导出、模板、缩放、模式切换
## 用户链路
**1. 创建文档**

用户进入产品后应立即看到三种入口：

- 新建空白文档
- 导入 Markdown
- 从模板创建

模板创建建议放在首屏，因为这是产品差异点。

**2. 输入内容**

输入内容时不要打断用户。

建议提供：

- Markdown 语法高亮
- 自动补全标题、列表、表格
- 图片拖入自动转 Markdown 引用
- 公式错误即时提示
- 右侧或底部显示解析状态

**3. 自动排版**

当用户输入内容后，系统自动生成分页预览。

需要明确状态：

- 正在解析
- 正在分页
- 已完成
- 存在排版问题

建议在画布上用轻提示，而不是弹窗。

**4. 套用模板**

模板套用建议做成强入口。

用户可以：

- 一键切换整套模板
- 预览模板效果
- 应用到当前文档
- 保留或覆盖手动样式

模板应用前建议提供选项：

- 保留我的局部调整
- 覆盖所有样式
- 仅应用页面和标题样式
- 仅应用页眉页脚

**5. 可视化微调**

微调不是要做成 Word，而是针对自动排版结果做有限控制。

优先支持：

- 插入分页符
- 设置段落不跨页
- 设置标题与下段同页
- 调整图片大小
- 调整表格列宽
- 设置元素前后间距
- 修改页眉页脚

**6. 导出检查与导出**

导出前建议必须出现检查面板。

检查项包括：

- 图片是否丢失
- 公式是否解析失败
- 是否有内容溢出页面
- 目录是否已更新
- 是否存在空白页
- 表格是否异常跨页
- 是否存在未保存修改

检查结果分三类：

- 错误：必须处理或确认忽略
- 警告：建议处理
- 提示：不影响导出

## 产品能力分层

为了避免功能混杂，把产品拆成五层能力。

**1. 内容层**

用户输入和管理原始内容。

- Markdown 原文
- 图片资源
- 公式源码
- 表格内容
- AI 生成内容
- 文档元信息

这一层强调输入效率，不强调排版。

**2. 结构层**

Markdown 被解析后的中间模型。

建议内部结构类似：

json

`{   "type": "heading",   "level": 2,   "id": "section-1",   "content": "函数的基本概念",   "styleRef": "heading-h2-default",   "metadata": {     "sourceRange": [120, 136],     "pageBreakBefore": false,     "keepWithNext": true   } }`

结构层的价值是：

- 支持目录生成
- 支持大纲导航
- 支持模板应用
- 支持 AI 理解文档结构
- 支持分页引擎计算
- 支持工程文件保存

**3. 样式层**

负责样式规则，而不是内容本身。

包括：

- 默认样式
- 文档模板
- 元素模板
- 用户局部样式
- 页面设置
- 页眉页脚设置

样式层要和内容层分离，否则后续模板系统会很难维护。

**4. 排版层**

负责从结构化内容生成分页结果。

包括：

- 页面尺寸
- 可用内容区域
- 元素测量
- 分页决策
- 溢出处理
- 页码计算
- 目录页码回填
- 导出布局稳定性

**5. 交互层**

用户实际操作的界面。

包括：

- Markdown 编辑
- 分页预览
- 右侧属性面板
- 浮动工具条
- 模板选择
- AI 面板
- 导出检查

# 功能设计
## 文件管理
### 1. 基础能力
- 新建文件夹
- 删除文件夹
- 新建 Markdown 文件
- 删除文件
- 文件/文件夹重命名
- 双击进入重命名

### 2. 文件类型建议

建议定义几类文件：

- `.md`：原始 Markdown 文件
- `.layout` 或 `.json`：结构化排版工程文件
- `.pdf`：导出文件
- 图片资源：本地或工程内资源

### 3. 保存机制

建议区分：

- 自动保存：每隔固定时间或内容变更后自动保存
- 手动保存：用户主动保存当前版本
- 版本快照：重要编辑节点生成历史版本
- 异常恢复：崩溃后恢复未保存内容

### 4. 拖拽细节

拖拽文件时需要明确：

- 文件可拖入文件夹
- 文件夹可拖入其他文件夹
- 不允许拖入自身或子级目录
- 拖拽时显示目标位置高亮
- 拖拽后自动保存目录结构

- 图片卡片模板

- 目录模板
- 便利贴模板
- 重点提示框模板

### 2. 套用规则

建议明确三种应用范围：

- 应用于当前选中元素
- 应用于同类元素
- 应用于全文

例如：

用户选中一个二级标题，套用「蓝色下划线标题」：

- 仅当前标题
- 所有二级标题
- 所有标题

### 3. 取消套用

支持：

- 当前元素恢复默认
- 同类元素恢复默认
- 全文恢复默认样式
- 恢复到上一个模板

### 4. 模板优先级

建议定义样式优先级：

`手动局部样式 > 元素模板 > 整套模板 > 默认样式`

这样用户手动调整的地方不会轻易被模板覆盖。

### 5. 模板保存

建议支持：

- 将当前文档样式保存为整套模板
- 将当前元素样式保存为元素模板
- 模板重命名
- 模板删除
- 模板分类
- 模板预览缩略图

## AI 生成辅助设计

原需求只写到「根据 markdown 内容/用户输入执行」，建议细化为 AI 面板。

### 1. AI 能力方向

**1. 生成内容**

适合空白页场景。

入口：

- 生成讲义
- 生成知识点总结
- 生成练习题
- 生成试卷初稿
- 根据大纲生成正文

结果处理：

- 插入当前文档
- 新建文档
- 仅复制 Markdown
- 应用推荐模板

**2. 优化内容**

适合选中文本场景。

入口：

- 润色
- 改写
- 总结
- 扩写
- 降低难度
- 提高正式度
- 转为课堂讲义风格
- 转为学生笔记风格

**3. 检查排版**

这是最符合产品定位的 AI 能力。

建议重点做：

- 检查标题层级是否混乱
- 检查段落是否过长
- 检查是否适合加入目录
- 检查图片是否缺少说明
- 检查表格是否过宽
- 检查分页是否不美观
- 推荐合适模板
- 建议哪里转成重点框
- 建议哪里插入小结或练习

AI 面板可以有一个明确按钮：

> 一键检查文档

输出结果类似：

- 第 2 页：图片过大，建议缩小至 80%
- 第 3 页：H2 标题位于页尾，建议与下段同页
- 第 5 页：表格较宽，建议横向页面或缩小字号
- 第 6 页：建议为本章节添加小结

用户可以逐条接受或忽略。

### 2. AI 交互方式

建议三种入口：

- 顶部 AI 按钮：打开 AI 面板
- 选中文本后悬浮菜单：润色、总结、扩写、改写
- 空白页面快捷入口：生成大纲、生成文档、生成试卷

### 3. AI 结果处理

AI 生成结果不应直接覆盖用户内容，建议提供：

- 插入到当前位置
- 替换选中内容
- 追加到文末
- 作为新文档打开
- 仅预览不应用
- 复制

### 4. AI 安全机制

建议支持：

- 用户确认后再应用
- 可撤销
- 保留原文版本
- 标记 AI 生成内容，后续可选
- 敏感内容提示，视业务需要

## 分页与排版引擎规则

**默认分页规则**

建议定义：

- 标题尽量不出现在页尾
- 标题默认与后一个段落保持同页
- 图片默认不拆分
- 图片过大时自动缩放到可用宽度
- 图片说明默认与图片保持同页
- 表格默认允许跨页
- 表格跨页时重复表头
- 公式默认不跨页
- 列表允许跨页，但避免单个列表项被拆得太碎
- 页眉页脚不占用正文区域之外的空间，而是参与版心计算

**用户控制项**

右侧属性面板中建议提供「分页控制」区域：

- 段前分页
- 段后避免分页
- 与下一段同页
- 当前元素不跨页
- 固定在当前页
- 允许跨页
- 跨页时重复标题/表头

这些控制项是产品专业感的重要来源。

**溢出处理策略**

当内容放不下时，系统应该有明确策略：

- 文本：自动流入下一页
- 图片：优先缩放，其次移至下一页
- 表格：优先跨页，其次提示异常
- 公式：优先整体移至下一页
- 便利贴：提示遮挡或超出
- 固定元素：超出时提示用户处理

# 交互设计详案

本章节定义 Layout2.0 的完整交互模型，涵盖用户从进入产品到完成导出全链路的每一个交互细节。交互设计遵循三项原则：**不打断用户的输入流**、**每个操作都有即时反馈**、**错误给出解决路径而非仅报障**。

---

## 一、工作区交互模型

### 1.1 三栏布局的交互规范

产品采用左（文件/大纲/搜索）、中（画布）、右（属性面板）三栏布局。交互规范如下：

**面板宽度**

- 左侧面板默认宽度 240px，支持拖拽分割线调整，范围 180px–400px
- 右侧面板默认宽度 280px，支持拖拽分割线调整，范围 220px–480px
- 调整后的宽度自动记忆，不随文档切换丢失
- 双击分割线：还原该面板到默认宽度

**面板显隐**

- 左面板：Ctrl/Cmd + B 切换显示/隐藏
- 右面板：Ctrl/Cmd + R 切换显示/隐藏
- 快捷键在两个面板同时隐藏时行为一致（再次按下恢复右面板）
- 隐藏面板时，画布区域无缝扩展填充空白区域

**Tab 切换**

- 左面板四个 Tab（文件、大纲、搜索、资源库）之间切换不保存滚动位置
- 右面板内容根据当前选中对象动态变化，无固定 Tab 结构

### 1.2 工具栏交互

顶部工具栏建议采用二级菜单结构，交互规范：

- 一级菜单项悬停 150ms 后展开二级面板（避免点击后立即展开导致鼠标滑入面板时误触）
- 二级面板展开动画：高度从 0 到内容高度，120ms ease-out
- 点击一级菜单以外的区域，或按下 Esc，一级菜单收起
- 工具栏图标按钮悬停时，tooltip 延迟 400ms 显示（避免鼠标快速划过时频繁闪烁）
- 当前不可用功能，图标置灰并显示禁用原因 tooltip
- 工具栏按钮建议按功能分组，每组之间留 8px 间隔，组内按钮紧邻

### 1.3 右键菜单

画布中不同对象支持上下文右键菜单：

| 选中对象  | 右键菜单项                                                         |
| ----- | ------------------------------------------------------------- |
| 文本段落  | 剪切 / 复制 / 粘贴 / 插入上方分页 / 设为标题 / 清除格式 / 查看样式来源                  |
| 图片    | 复制 / 剪切 / 删除 / 替换图片 / 设为左环绕 / 设为右环绕 / 设为居中 / 锁定比例 / 重置尺寸      |
| 表格    | 剪切 / 复制 / 插入行 / 插入列 / 删除行 / 删除列 / 合并单元格 / 拆分单元格 / 设置表头 / 允许跨页 |
| 公式    | 复制 LaTeX / 编辑 / 删除 / 设为编号公式                                   |
| 页面空白区 | 插入分页符 / 插入图片 / 插入表格 / 页面设置                                    |
| 多选元素  | 对齐（顶对齐/居中/底对齐/左对齐）/ 分布 / 组合 / 置于顶层 / 置于底层                     |

右键菜单出现时，距鼠标指针 8px 偏移（避免遮挡触发点）。菜单项按操作频率排序，常用操作置顶。

## 二、视图切换交互

### 2.1 三种视图的定义与切换

产品提供三种视图，定义如下：

**源码视图**

- 全屏左侧面板隐藏，只保留工具栏 + 源码编辑区 + 底部状态栏
- 编辑器区占满整个可用宽度
- 右侧属性面板自动隐藏（源码模式下无选中对象，上下文无意义）
- 适合场景：纯内容输入、长文档快速编写

**分屏视图**

- 左：Markdown 源码编辑区（默认 50% 宽度）
- 右：分页预览画布（默认 50% 宽度）
- 源码编辑区和画布同步滚动（源码区滚动时，画布跟随定位到对应内容所在页面）
- 支持拖拽分割线调整左右比例，范围 20%–80%
- 适合场景：边写边调排版

**预览视图**

- 全屏画布，工具栏 + 左面板（可选） + 画布 + 右面板（选中元素时）
- 适合场景：专注排版、整体审阅、最终导出前检查

### 2.2 视图切换交互

切换方式：

- 工具栏视图下拉菜单
- 底部状态栏视图切换快捷按钮（三个图标按钮）
- 快捷键：Ctrl/Cmd + 1（源码）、Ctrl/Cmd + 2（分屏）、Ctrl/Cmd + 3（预览）

切换动画：视图切换时，内容区域采用横向滑动过渡，150ms ease-in-out。源码视图切换到分屏视图时，左侧编辑区从当前宽度逐步扩展，右侧画布同步淡入。

### 2.3 编辑器与画布联动

分屏视图下，编辑器与画布之间的联动交互：

**滚动联动**

- 源码编辑区向下滚动时，画布跟随定位到对应内容区域
- 画布手动滚动时，源码编辑区自动跟随（定位到该页对应的源码位置）
- 联动可关闭：画布底部提供"仅手动滚动"开关

**焦点联动**

- 鼠标点击源码编辑区，焦点指示器显示当前行在画布中对应的页面位置
- 鼠标点击画布某页，源码编辑区自动滚动到该页内容对应的位置（高亮闪烁 300ms 提示）

**编辑联动**

- 在源码中输入内容，实时在画布对应位置渲染
- 渲染延迟不超过 300ms，超出时在画布底部显示"正在渲染..."微调提示
- 在画布中选中图片并拖拽调整大小，自动同步更新源码中的尺寸参数

## 三、画布交互细节

### 3.1 缩放交互

缩放控制位于画布底部工具条：

- 缩放范围：25%–400%
- 步进：25%
- 预设快捷：适应宽度、适应高度、100%
- 交互方式：
  - 鼠标滚轮 + Ctrl/Cmd：缩放（以鼠标位置为中心）
  - 缩放滑块：拖拽即时响应
  - 缩放数字按钮：点击后变为可输入框，直接输入百分比值回车确认
- 缩放时保持当前视口中心点不变
- 缩放状态自动记忆，切换文档时恢复上一次的缩放比例

### 3.2 页面滚动

- 单页视图：垂直滚动，页面从上往下依次排列，间距 20px
- 双页视图：模拟书籍翻页效果，偶数页在左、奇数页在右；支持键盘左右方向键翻页
- 快速定位：画布右下角显示"页面导航器"缩略图，点击缩略图任意位置跳转到对应页
- 页面导航器支持拖拽红框指示当前视口位置

### 3.3 页面选择

- 单击页面空白处：选中该页面，右侧面板显示页面设置
- 单击页面内元素：选中该元素，忽略页面选中状态
- 按住 Shift + 单击：多选页面（用于批量设置页面属性）
- 按住 Ctrl/Cmd + 单击：切换单个页面的选中状态
- 页面选中时，页面边框高亮显示（2px 蓝色边框）

### 3.4 元素选择与操作

**选中状态**

- 单击元素：选中该元素，显示 8 个调整手柄（四个角 + 四条边中点）
- 双击文本段落：进入文本编辑模式，光标定位到点击位置
- 选中元素后 200ms 内，再次单击画布空白处取消选中

**多选**

- 按住 Shift + 依次点击：多选多个元素
- 画布左上角提供"框选"工具（快捷键 M），拖拽矩形框选中框内所有元素
- 多选时，调整手柄变为批量操作手柄（对齐、分布、组合）

**拖拽调整**

- 拖拽元素四条边中点的手柄：沿单个方向调整尺寸
- 拖拽元素四角手柄：等比缩放（默认），按住 Shift 解除比例锁定
- 按住 Alt 拖拽：从元素中心缩放而非从角落
- 拖拽时显示尺寸参考线（当前宽高数值实时显示在元素旁边）
- 拖拽过程中元素半透明显示（opacity: 0.7），释放后恢复正常
- 元素移动时，自动吸附到页面边界、页边距线、页面中线、相邻元素边缘（吸附阈值 5px）
- 按住 Ctrl 拖拽可临时禁用吸附

### 3.5 辅助线与网格

- 默认显示页边距辅助线（虚线，灰色）
- 开启"对齐辅助线"后，元素拖拽时自动显示与其他元素的对齐参考线
- 辅助线颜色建议：红色（页面边界）、蓝色（其他元素边缘对齐）、绿色（间距相等）
- 显示/隐藏辅助线：画布底部工具条快捷开关

## 四、内容编辑交互

### 4.1 Markdown 编辑器交互

**基础编辑**

- 语法高亮：实时解析并渲染，解析错误行左侧显示红色波浪线，hover 显示错误原因
- 自动续写：输入 `1.` 后回车，自动生成 `2.`；输入 `- ` 后回车，自动生成下一个列表项
- Tab 键：在列表中按 Tab 缩进为二级列表，Shift+Tab 取消缩进
- 输入 `#` 后空一格再输入文字，自动识别为一级标题，右侧实时渲染预览
- 输入 ` ``` ` 后直接回车，自动补全代码块结束标记并换行

**图片拖入**

- 将图片文件从系统拖入编辑器区域，显示放置预览区（半透明蓝色边框区域）
- 释放后自动生成 Markdown 图片语法，图片保存到工程资源目录
- 支持批量拖入多张图片，依次插入

**公式交互**

- 输入 `$` 进入行内公式编辑模式，左侧编辑 LaTeX，右侧实时预览
- 输入 `$$` 加回车，进入独立公式编辑模式
- 公式编辑完成后，点击编辑器外区域或按 Esc 退出编辑
- 公式解析失败时，显示红色边框并在行尾显示错误图标，hover 显示具体错误信息

### 4.2 浮动工具条

选中文本后，显示浮动工具条，交互规范：

- 显示位置：选中文字上方 8px 处，水平居中于选中区域
- 显示动画：从 0 到 100% 透明度，100ms
- 工具条内容：加粗、斜体、下划线、删除线、字体颜色、高亮、链接、清除格式
- 工具条按钮悬停时显示 tooltip 说明（延迟 400ms）
- 选中文字样式已应用的按钮，显示激活状态（深色背景）
- 点击画布空白处，工具条消失（150ms 淡出）
- 选中文字后，右侧属性面板同步显示详细样式设置

### 4.3 表格交互

**插入表格**

- 输入 `/table` 唤起命令面板，或工具栏插入 → 表格
- 弹出表格设置浮层：选择行数（1–20）和列数（1–10）的网格
- 点击确认后，表格以最小尺寸（每格 50px 宽）插入

**表格内编辑**

- 单击单元格：选中该单元格，光标进入编辑模式
- 双击单元格：选中文本或进入编辑（已有内容时）
- Tab 键：跳转到下一单元格（末尾单元格按 Tab 新增一行）
- Enter 键：单元格内换行
- Shift + Tab：跳转到上一单元格
- 方向键：单元格内文本导航

**表格结构调整**

- 右键单元格 → 插入行/列、删除行/列
- 选中单元格后，表格边缘显示调整手柄（行高调整手柄在行与行之间，列宽调整手柄在列与列之间）
- 拖拽列分隔线调整列宽，实时预览
- 双击列分隔线：还原为"自动列宽"（根据内容自动计算）
- 右键单元格 → 合并单元格 → 选择合并范围（弹出网格选择）

**表格跨页处理**

- 表格选中时，右侧面板提供"跨页设置"区域：
  - 允许跨页（默认开启）
  - 表头在每页重复
  - 表尾在每页重复
  - 不允许拆分为两页（整表强制移至下一页）
- 表格自动跨页时，在断页处显示断页提示线（虚线），鼠标悬停显示"此处跨页"提示

## 五、模板交互流程

### 5.1 模板发现与预览

模板入口设计：

- 工具栏"模板"菜单 → "浏览全部模板"
- 打开模板库面板（覆盖画布区域的模态面板）
- 模板以卡片网格展示，每张卡片显示：缩略图、模板名称、适用场景标签
- 鼠标悬停模板卡片：自动放大预览（scale 1.05），显示"预览效果"浮层
- 预览浮层显示该模板的 2–3 页典型内容样例

### 5.2 模板应用交互

应用模板时，建议分三步交互：

**第一步：选择模板**

- 在模板库中单击选中模板，被选中模板卡片显示蓝色边框
- 点击"应用到当前文档"按钮

**第二步：选择覆盖范围**

弹出"应用模板"确认面板，提供四个选项（单选）：

- 保留我的局部调整（仅应用页面设置 + 整体样式，保留用户手动修改过的元素样式）
- 覆盖所有样式（全文应用模板，不保留任何手动样式）
- 仅应用页面和标题样式（不影响正文、列表等其他样式）
- 仅应用页眉页脚（仅替换页眉页脚，不影响其他样式）

**第三步：确认与预览**

- 点击确认前，提供"预览效果"按钮：在新标签页/新窗口中打开应用模板后的效果预览（不实际修改文档）
- 预览确认无误后，点击"应用"，文档更新，画布刷新

### 5.3 模板保存交互

将当前文档保存为模板：

- 工具栏"模板" → "保存为模板"（或在右侧样式面板中点击"另存为模板"）
- 弹出保存模板浮层：
  - 模板名称（必填，自动填充当前文档名作为建议名）
  - 适用场景（多选标签：讲义/试卷/笔记/论文/教辅）
  - 模板说明（可选，描述模板特点）
  - 缩略图（自动截取当前文档首页，裁剪为模板缩略图）

├─────────────────────────────────┤

│  （流式输出，实时显示）           │
├─────────────────────────────────┤
│  [插入当前文档] [新建文档] [复制] │  ← 操作按钮
└─────────────────────────────────┘
```

### 6.2 三种 AI Tab 的交互设计

**生成 Tab**

- 输入区上方显示"快速生成"快捷卡片：一键生成讲义 / 练习题 / 知识点总结 / 试卷初稿

- 生成过程中，输入区显示"生成中..."动画，生成结果以流式方式逐段显示在结果区
- 生成完成后，操作按钮激活：插入当前文档 / 新建文档 / 复制 Markdown / 仅预览

**优化 Tab**

- 选中文档中部分内容后，"优化 Tab"自动激活并显示选中文字预览
- 提供优化选项：润色 / 改写 / 总结 / 扩写 / 降低难度 / 提高正式度 / 转为课堂讲义风格
- 选择优化类型，点击"开始优化"
- 优化结果与原文并排对比显示（左侧原文，右侧优化结果）
- 差异高亮显示（新增内容绿色背景，删除内容红色删除线）
- 操作按钮：替换选中内容 / 插入到光标处 / 取消

**检查 Tab**

- 点击"一键检查文档"，系统分析当前文档结构与排版
- 检查过程中，页面左侧显示检查进度条："正在检查标题层级 → 正在检查图片尺寸 → 正在检查分页美观度..."
- 检查完成后，显示检查结果列表：

```
✅ 第 1 页：标题层级正确
⚠️ 第 3 页：H2 标题位于页尾，建议与下段同页
⚠️ 第 5 页：表格宽度接近页面宽度，建议缩放至 85%
✅ 第 7 页：目录已更新
⚠️ 第 8 页：建议为本章节添加小结
❌ 第 2 页：图片 image-001.png 路径失效
```

- 每条结果右侧提供"应用建议"按钮（一键修复）或"忽略"按钮
- "全部忽略"和"全部应用"批量操作位于列表顶部

### 6.3 AI 交互安全

- AI 操作前必须用户确认（显示预览 → 用户点击确认 → 执行）
- 所有 AI 操作自动记录到撤销栈，支持一键撤销
- AI 生成的内容自动在源码中添加注释标记 `<!-- AI生成内容 -->`（可关闭）
- 长时间 AI 操作（超过 5 秒）显示取消按钮，用户可随时终止

### 7.1 全局状态指示器

画布底部固定状态栏（高度 28px），从左到右显示：

| 位置 | 显示内容 |
|---|---|
| 左 | 当前视图模式（源码/分屏/预览） |

| 最右 | 保存状态：💾 已保存 / ⏳ 未保存 / 🔄 保存中 |

### 7.2 Toast 提示规范

轻提示（Toast）用于非阻塞式操作反馈：

- 出现位置：画布右下角，距边缘 20px
- 自动消失时间：错误类 8 秒，成功类 3 秒，提示类 4 秒
- 用户可将鼠标悬停在 Toast 上延长显示时间（防止误操作快速消失）
- 最多同时显示 3 条 Toast，超出时旧提示依次向上推移
- 成功提示绿色，警告提示橙色，错误提示红色
- 错误 Toast 右侧提供"查看详情"链接，点击展开错误信息和解决建议

### 7.3 模态对话框规范

阻塞式对话框用于不可逆或高影响操作：

- 居中显示，背景遮罩层 opacity 0.4
- 弹出动画：从 scale(0.95) opacity(0) 过渡到 scale(1) opacity(1)，150ms
- 关闭方式：点击确定/取消按钮、点击遮罩层、按 Esc
- 对话框最大宽度 480px，超长内容内部滚动
- 按钮顺序：主操作按钮在右（蓝色填充），次要操作按钮在左（灰色边框）

需要弹出确认对话框的场景：

- 导出前检查（发现问题时）
- 删除文件/文件夹
- 模板覆盖全部样式
- 关闭有未保存修改的文档
- 清空文档内容

### 7.4 进度指示

长耗时操作（解析、导出）使用进度指示：

- 底部状态栏显示进度条（细线型，进度条颜色 #3B82F6）
- 进度条下方显示当前步骤文字："正在导出 PDF（3/5）：压缩图片..."
- 支持取消：进度条右侧显示 × 按钮，点击终止操作
- 导出完成后，进度条变为绿色完成状态，2 秒后自动消失

# 技术选型方案

## 一、选型方法论

技术栈的选择由产品的核心能力反向驱动。以下按优先级列出每个能力模块对技术的硬性约束，作为选型依据。

| 优先级 | 核心能力 | 技术约束 |
|---|---|---|
| P0 | 分页排版渲染引擎 | 精确控制像素级布局，支持分页、页眉页脚、溢出处理 |
| P0 | 跨平台桌面应用 | Windows/macOS/Linux 全平台分发 |
| P0 | PDF 导出 | 渲染结果必须能精确转换为 PDF |
| P1 | Markdown 编辑器 | 实时语法解析、错误提示、自动补全 |
| P1 | 模板系统 | 样式与内容分离，可序列化/反序列化 |
| P1 | 结构化文档模型 | JSON 作为中间层，支撑大纲、目录、AI 分析 |
| P2 | AI 集成 | 与大语言模型 API 对接 |
| P2 | 图片/公式渲染 | KaTeX 公式渲染、图片资源管理 |
| P3 | 触控/平板适配 | 后续考虑，不影响 MVP 选型 |

## 二、技术域评估

### 2.1 桌面框架（架构设计为未来网页版留足空间）

**候选方案对比**

| 维度      | Electron               | Tauri          | Flutter     |
| ------- | ---------------------- | -------------- | ----------- |
| 语言      | JavaScript/TypeScript  | Rust + Web     | Dart        |
| 成熟度     | 极高（VS Code、Fiquid 等）   | 高（快速增长）        | 中（桌面端相对新）   |
| 生态      | npm 生态极其丰富             | Rust 生态增长中     | Flutter 生态  |
| 包体积     | 较大（~150MB）             | 极小（~3MB）       | 中等（~50MB）   |
| PDF 生成  | 可直接调用 Chromium 打印 API  | 需配合 Rust PDF 库 | 需自建方案       |
| 原生能力    | 好                      | 极好             | 好           |
| 学习成本    | 低（前端团队友好）              | 中（需学 Rust）     | 中（需学 Dart）  |
| 分页引擎契合度 | 高（Chromium 渲染层可直接用于打印） | 中（需自行实现打印管线）   | 低（自定义渲染成本高） |

**结论：Electron + Vite + React**

理由：

- 分页排版引擎需要在渲染层实现精确分页控制，Electron 内置的 Chromium 渲染引擎与 PDF 打印 API 可无缝对接，渲染层代码可直接复用于打印输出，保证所见即所得的保真度
- 前端团队友好，React 生态中的编辑器库（CodeMirror/Monaco）、排版库（TypeScript）可直接使用，无需重新造轮子
- Tauri 包体积小但 PDF 打印管线需要自行构建 Rust 层，增加不确定性
- Flutter 桌面端在中文排版和复杂排版控制上积累不如 Electron 深厚

### 2.2 渲染引擎

**核心矛盾**：浏览器默认的分页算法（CSS Paged Media / `@page`）能力有限，无法满足"标题不落页尾"、"表头跨页重复"、"元素溢出处理"等精细控制。

**候选方案对比**

| 维度     | CSS Paged Media + polyfill | 自研 Canvas/SVG 引擎 | PDF.js 渲染层   |
| ------ | -------------------------- | ---------------- | ------------ |
| 分页控制粒度 | 粗（只能按块分页）                  | 细（逐行逐元素测量）       | 细            |
| 与编辑器联动 | 易                          | 易                | 难（文档结构需双向同步） |
| 所见即所得  | 好                          | 好                | 中（需额外映射层）    |
| 开发成本   | 低（依赖浏览器实现）                 | 高（完全自研）          | 中            |
| 模板样式复用 | 中                          | 好                | 中            |

**结论：自研排版引擎（TypeScript）+ PDFTron/CDK（辅助）**

理由：

- 本产品的核心竞争力在于专业级排版控制——表头跨页重复、标题与下段同页、精确行高计算等，这些都需要逐元素测量和分页决策，CSS Paged Media 无法满足
- 排版引擎作为独立模块（TypeScript），与编辑器层（CodeMirror）和渲染层（Canvas/DOM）分别通过结构化 JSON 解耦，便于后续优化和测试
- 导出层直接调用 Electron 的 `webContents.printToPDF()` API，渲染引擎的输出 DOM 树即为打印输入，保证导出保真度
- MVP 阶段优先实现 DOM 渲染（直接复用 CSS 排版），在 MVP 验证后再迁移到 Canvas 精确控制层，分阶段降低风险

### 2.3 Markdown 解析

| 维度 | marked | markdown-it | remark（unified生态） |
|---|---|---|---|
| 插件生态 | 一般 | 丰富 | 极丰富（rehype/retext） |
| AST 质量 | 中 | 中 | 高（mdast + hast 双 AST） |
| 插件开发体验 | 一般 | 好 | 极好（管线式） |
| LaTeX/表格支持 | 需插件 | 需插件 | remark-math + remark-gfm 成熟 |
| 与排版引擎联动 | 一般 | 一般 | 好（可直接输出 hast 供渲染） |
| 性能 | 快 | 快 | 中（更通用，体积略大） |

**结论：remark（unified 生态）**

理由：

- `remark` 生态的 `mdast` → `remark-plugin` → `rehype` 管线天然匹配"Markdown → 结构化 JSON → DOM 渲染"三层架构
- `remark-math` + `remark-gfm` 处理公式和表格开箱即用
- `rehype` 输出 `hast`（HTML AST）后，可直接交由排版渲染层消费
- 插件体系完善，未来扩展（如插入自定义块）成本低

### 2.4 编辑器内核

| 维度            | CodeMirror 6 | Monaco Editor | ProseMirror      |
| ------------- | ------------ | ------------- | ---------------- |
| 体积            | 轻量（<200KB）   | 重（>5MB）       | 中                |
| Markdown 语法高亮 | 需配置（现成扩展）    | 内置            | 需配置              |
| 自动补全          | 扩展支持         | 内置            | 扩展支持             |
| 与 DOM 渲染联动    | 易（DOM 回填）    | 难（需额外映射）      | 易                |
| 协同编辑          | Y.js 集成成熟    | Y.js 集成       | ProseMirror 原生支持 |
| API 灵活性       | 高            | 中             | 高                |

**结论：CodeMirror 6**

理由：

- 体积轻量，适合嵌入 Electron 的侧边编辑区
- Markdown 扩展（`@codemirror/lang-markdown`）开箱即用，高亮效果优秀
- 编辑器内容与排版引擎通过 JSON AST 解耦，CodeMirror 的 `DocChange` 事件触发解析管线
- Monaco 太重（VS Code 同款），且不适合作为嵌入组件；ProseMirror 上手成本略高

### 2.5 公式渲染

| 维度          | KaTeX            | MathJax 3      |
| ----------- | ---------------- | -------------- |
| 渲染速度        | 极快（WebWorker 异步） | 较快             |
| 包体积         | 小（~200KB）        | 大（~300KB）      |
| LaTeX 兼容性   | 较好（常用数学符号）       | 好（全 LaTeX）     |
| 与 remark 集成 | remark-math 插件   | remark-math 插件 |
| 错误提示        | 有限               | 好              |
| 导出 PDF      | 好（生成精确 MathML）   | 好              |

**结论：KaTeX**

理由：

- 渲染速度是用户体验的关键，KaTeX 比 MathJax 快 2–3 倍
- 包体积更小，适合 Electron 打包
- 与 remark-math 插件集成成熟
- 对于教育场景的常用数学符号覆盖完整

### 2.6 状态管理

| 维度 | Zustand | Redux Toolkit | Jotai |
|---|---|---|---|
| 复杂度 | 低 | 中 | 中 |
| 性能 | 好 | 好 | 好 |
| 撤销/重做支持 | 插件（zustand-middleware-undo） | middleware | 需自建 |
| 与排版引擎联动 | 好 | 好 | 好 |
| TypeScript 支持 | 好 | 极好 | 好 |

**结论：Zustand + Immer**

理由：

- Zustand API 简洁，学习成本低，适合中型前端团队
- 内置 `subscribeWithSelector`，排版引擎和编辑器可独立订阅状态变更
- `zustand/middleware/immer` 支持不可变更新，调试友好
- 撤销/重做通过中间件实现，粒度可控（编辑器层和排版层分开记录）

### 2.7 项目构建

| 维度 | Vite | webpack | tsup |
|---|---|---|---|
| 开发体验 | 极好（HMR 快） | 一般 | 快（仅打包） |
| Electron 主进程构建 | electron-vite 成熟 | electron-builder | 需自行组合 |
| TypeScript | 原生支持 | 需要配置 | 原生支持 |
| 生态 | 极好 | 极好 | 一般 |

**结论：Electron + Vite（electron-vite）**

理由：

- `electron-vite` 是 Electron + Vite 的最佳实践，整合了主进程、渲染进程、预加载脚本的构建
- HMR 速度快，适合快速迭代
- 与 React + TypeScript 集成成熟

## 四、MVP 技术优先级

考虑到 MVP 阶段需要快速验证产品核心价值，技术投入分以下优先级：

**P0 — MVP 必须实现**

- Electron + Vite + React + TypeScript 基础架构
- CodeMirror 6 Markdown 编辑器
- remark + rehype Markdown → hast 解析管线
- DOM 层排版渲染（CSS + Flexbox/Grid 布局）
- 分页逻辑（基础版：按 DOM 块分页）
- PDF 导出（Electron PrintToPDF）
- Zustand 状态管理 + 基础撤销重做
- 基础模板系统（JSON 序列化模板）
- 文件管理（新建/打开/保存/另存为）

**P1 — MVP 后第一迭代**

- 精细化排版引擎（逐元素测量、标题孤寡行控制、表格跨页）
- 表格可视化编辑（选中/拖拽调整）
- 图片环绕布局
- 公式渲染（KaTeX）
- 目录自动生成与跳转
- 页眉页脚系统
- 大纲面板

**P2 — 产品完善**

- AI 面板（生成/优化/检查）
- 命令面板
- 多视图切换（源码/分屏/预览）
- 新手引导
- 模板库系统
- 搜索与替换

**P3 — 体验增强**

- 触控/平板适配
- Canvas 精确渲染层
- 协同编辑
- 插件系统

## 五、技术风险与应对

| 风险                        | 概率  | 影响  | 应对策略                                         |
| ------------------------- | --- | --- | -------------------------------------------- |
| 自研排版引擎复杂度高，周期不可控          | 高   | 高   | MVP 阶段先用 DOM + CSS 分页，精细控制作为第二迭代重点           |
| Electron 包体积大（~150MB）     | 中   | 低   | 使用 electron-builder 的压缩选项，macOS 可用 `.pkg` 分发 |
| CodeMirror 6 与自研渲染层联动调试成本 | 中   | 中   | 设计明确的接口协议（Document State），双向同步通过事件驱动解耦       |
| remark 插件链调试困难            | 低   | 中   | 拆分为独立解析模块，单元测试覆盖每个 pipeline 环节               |
| AI API 成本与响应速度            | 中   | 中   | 设计本地缓存层，AI 结果可复用；流式输出优化感知速度                  |
